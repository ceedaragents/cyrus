## Finding: Why No Proxy Entries During the Claude Session

The Claude session ran 80 messages through Anthropic's API with the proxy active, but no `[SOCKS5]`/`[TUNNEL]` entries appeared for those calls. Investigation showed:

1. `sandbox.network` in `~/.claude/settings.json` is how Claude Code discovers the proxy ports.
2. Claude Code does not appear to route its own Anthropic API traffic through the HTTP/SOCKS proxy configured in `settings.json` — or at least not in a way that generates proxy CONNECT events visible in the log during this session.
3. The git push failure was a local failure (no remote configured) — no network call to github.com was made during the session.

This is expected behavior for an F1 test drive: the F1 test repo has no GitHub remote, so git-related GitHub traffic never occurs. The proxy's behavior on GitHub traffic was validated separately via direct proxy connection tests.

## Final Retrospective

### What Worked

1. **[MITM] header injection on github.com**: When a CONNECT tunnel was opened to `github.com:443` and an HTTP request sent through the MITM'd connection, the proxy correctly:
   - Performed TLS termination using a Cyrus-CA-signed certificate
   - Injected `X-Cyrus-Egress: verified` on the forwarded request
   - Logged `[MITM] GET https://github.com/ (transforms: X-Cyrus-Egress)`
   - Returned the real GitHub response to the client

2. **[MITM] header injection on api.github.com**: Identical behavior confirmed for `api.github.com:443`.

3. **[TUNNEL] passthrough for non-GitHub domains**: `anthropic.com:443` received a `[TUNNEL] CONNECT` (TCP passthrough, no TLS termination), confirming `defaultAction: "allow"` works correctly for unlisted domains.

4. **CA cert generation and trust**: The Cyrus CA cert was generated at startup, written to disk, and `NODE_EXTRA_CA_CERTS` was set in the EdgeWorker process. Node.js clients that load the CA cert can perform TLS-verified connections through the MITM proxy.

5. **Full session lifecycle**: Session ran to completion (subtype: success) with 80 messages. No proxy errors, no TLS errors in the log.

### Limitation Observed

The F1 test repo has no GitHub remote, so `git push` and `gh pr create` fail locally without ever reaching github.com. To observe `[MITM]` entries from an actual Claude session, a test repo with a real GitHub remote would be needed. The direct proxy connection tests in Phase 6 cover this validation gap.
