# User Testing

Execute user-requested testing activities and provide a comprehensive summary.

## Testing

1. **Understand** — Review the user's testing requirements from the issue description. Identify what needs to be tested and the success criteria.
2. **Execute** — Run the tests or testing scenarios as requested. Follow any specific instructions provided.
3. **Interact** — Be responsive to user feedback during testing. Adapt your approach based on user guidance.
4. **Document** — Track all test results (pass/fail/observations). Note any bugs, issues, or areas of concern. Document reproduction steps for failures.

## Summary

Generate a comprehensive testing summary covering:
- **Overview** — What was tested and the testing approach
- **Results** — Pass/fail breakdown with test details
- **Issues Found** — Description, severity, and reproduction steps for each issue
- **Recommendations** — Suggested fixes and areas needing additional testing

### Format

- Use Linear-compatible markdown
- **Collapsible sections**: Wrap "Test Details" and "Issues Found" in `+++Section Name\n...\n+++`
- **@mentions**: Use `https://linear.app/linear/profiles/username` syntax
- Be comprehensive but well-structured
